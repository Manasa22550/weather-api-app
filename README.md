# Live Weather Dashboard — API Integration

## Overview
This project is a dynamic, responsive web application built as part of Task 3 of the InternSpark Web Development Internship. The core engineering focus of this project is third-party API integration, native asynchronous JavaScript workflow execution, and real-time JSON response mapping to inject live data parameters directly into a browser client interface.

## Project Objectives
* **Asynchronous API Consumption:** Implement modern JavaScript asynchronous architecture (`async/await` blocks) and standard browser network pipelines (`fetch()`) to retrieve server-side data from an external API weather service mesh.
* **Dynamic Client-Side Mutation:** Map raw API JSON object fields to dynamically populate structural HTML target DOM elements.
* **Defensive Code Architecture:** Construct robust `try/catch` safety gates and fallback database endpoints to defend against activation delays and unexpected network request drops.
* **Modern Interface presentation:** Create a user-friendly layout utilizing a responsive linear-gradient backdrop, a polished glassmorphism data card structure, and clean CSS utility visibility rules.

## Core Features
* **Live Environmental Fetching:** Queries a global geolocation tracking and weather forecasting mesh in real time based on user-typed parameters.
* **Dynamic Condition Mapping:** Features structural parsing dictionaries that translate numerical database condition status integers into readable strings accompanied by descriptive emoticons (e.g., Clear Sky ☀️, Moderate Rain 🌧️).
* **Fault-Tolerant Feedback System:** Toggles active visibility wrapper elements (`.hidden`) to slide layout metrics into view smoothly on successful operations while showing descriptive alerts during invalid queries.
* **Multi-Input Listening Bridges:** Hooks event triggers to listen for both graphical click behaviors and instant keyboard `Enter` actions.

## Technologies Implemented
* **Logic Brain:** Vanilla JavaScript (ES6 asynchronous networking and error catch pipelines)
* **Visual Formats:** CSS3 layout formatting blocks and keyframe tracking
* **Structure:** HTML5 Semantic markup
* **Cloud Infrastructure Hosting:** GitHub Pages Production Framework
* **Environment Control:** Visual Studio Code & GitHub Desktop

## File Architecture Directory
* `index.html` — Houses user inputs, search buttons, layout skeletons, and data injection nodes.
* `style.css` — Standardizes modern aesthetic sheets, flex arrangements, and transition layers.
* `script.js` — Controls the fetch engine pipeline, data encoder parameters, validation logic, and element injection maps.
